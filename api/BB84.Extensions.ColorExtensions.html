Remarks

This class includes methods to create Color objects from ARGB or RGB byte arrays and hex strings, as well as methods to convert Color objects back into these formats. It supports both 32-bit ARGB and 24-bit RGB representations. The methods handle endianness where applicable and validate input to ensure correct usage.

FromARGBHexString(string)

Converts a hexadecimal ARGB color string to a Color object.

public static Color FromARGBHexString(this string value)

Parameters

value string

A string representing the color in ARGB hexadecimal format.

Returns

Color

A Color instance corresponding to the specified ARGB hexadecimal string. Returns Empty if the input string does not meet the required format.

Remarks

The input string must be in the format "#AARRGGBB", where:

  • AA represents the alpha channel (transparency).
  • RR represents the red channel.
  • GG represents the green channel.
  • BB represents the blue channel.
Example: "#FF112233" represents a fully opaque color with red = 17, green = 34, and blue = 51.

FromArgbByteArray(byte[])

Creates a Color instance from a byte array containing ARGB color components.

public static Color FromArgbByteArray(this byte[] value)

Parameters

value byte[]

A byte array containing exactly 4 elements, representing the alpha, red, green, and blue components of the color in ARGB order.

Returns

Color

A Color instance corresponding to the specified ARGB components.

Remarks

The byte array must represent the color components in ARGB order, with each component occupying one byte. If the system architecture is not little-endian, the byte array is reversed before processing.

Exceptions

ArgumentException

Thrown if value does not contain exactly 4 elements.
